## Honest verdict

- **The mechanism works end-to-end** (separately covered by unit tests: PreCompact
  snapshot of branch/dirty/commits → SessionStart re-injection, staleness and
  wrong-session guards).
- **Differentiation on this fixture is real but modest**: equipped agents went straight
  to the dirty file instead of surveying (−19% tool calls and −12% tokens on haiku;
  −13% / −2% on sonnet). Nobody redid done work — on a 4-file repo with a high-signal
  test suite, the suite itself reveals migration state in one read, so the snapshot's
  informational edge is small *here*.
- **Where the snapshot should matter more** (future eval, larger fixture): big repos
  where surveying costs thousands of tokens; in-flight work the test suite can't see
  (docs, config, data files); multi-repo or detached-HEAD states; and dirty files far
  from the task's obvious entry point.

## Design note

Equipped prompts mirror the exact `renderHandoff()` output — what the SessionStart hook
injects in production — so this eval doubles as a fidelity check on that wording.
